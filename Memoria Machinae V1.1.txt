Memoria Machinae v1.1

Production-grade episodic memory microservice for embodied and language agents.

Changes in v1.1 over v1.0:

Replaced CLIP with SigLIP (google/siglip-base-patch16-224) for improved vision-language alignment

Replaced en_core_web_sm with en_core_web_trf (transformer-based NER) plus a robotics domain vocabulary for objects, actions, locations, and sensors

Added BlackBoxQueryEngine for forensic log querying (filter by subject, operation, time range)

Added unit and integration test suite (tests/) covering EventFilter, FeedbackIngester, RecallEngine, and consolidation logic

All references to prior project lineage removed; Memoria Machinae is the canonical name

# scheduler.py """ Background maintenance scheduler. Runs two recurring jobs per registered MemoriaStack: 1. decay_job — applies long-term strength decay to all nodes 2. consolidation_job — merges near-duplicates, prunes weak nodes, runs dream() Uses APScheduler's AsyncIOScheduler so jobs share the FastAPI event loop without blocking the request path. """ import logging from apscheduler.schedulers.asyncio import AsyncIOScheduler from config import CONFIG from metrics import BACKGROUND_DECAY_COUNT, BACKGROUND_CONSOLIDATION_COUNT logger = logging.getLogger(__name__) class MemoryScheduler: def __init__(self): self._scheduler = AsyncIOScheduler() self._registered: set[str] = set() def register(self, stack) -> None: subject_id = stack.subject_id if subject_id in self._registered: return self._scheduler.add_job( self._decay_job, trigger='interval', seconds=CONFIG['decay_interval_seconds'], args=[stack], id=f'decay_{subject_id}', replace_existing=True, ) self._scheduler.add_job( self._consolidation_job, trigger='interval', seconds=CONFIG['consolidation_interval_seconds'], args=[stack], id=f'consolidation_{subject_id}', replace_existing=True, ) self._registered.add(subject_id) logger.info( f"[SCHEDULER] Registered background jobs for subject '{subject_id}'. " f"Decay every {CONFIG['decay_interval_seconds']}s, " f"consolidation every {CONFIG['consolidation_interval_seconds']}s." ) async def start(self) -> None: self._scheduler.start() logger.info("[SCHEDULER] Background scheduler started.") async def stop(self) -> None: self._scheduler.shutdown(wait=False) logger.info("[SCHEDULER] Background scheduler stopped.") @staticmethod async def _decay_job(stack) -> None: subject_id = stack.subject_id try: stack.graph.decay_all() BACKGROUND_DECAY_COUNT.labels(subject_id=subject_id).inc() logger.debug(f"[SCHEDULER] Decay pass complete for '{subject_id}'.") except Exception as e: logger.error(f"[SCHEDULER] Decay job failed for '{subject_id}': {e}") @staticmethod async def _consolidation_job(stack) -> None: subject_id = stack.subject_id try: await stack.consolidate() BACKGROUND_CONSOLIDATION_COUNT.labels(subject_id=subject_id).inc() logger.debug(f"[SCHEDULER] Consolidation pass complete for '{subject_id}'.") except Exception as e: logger.error(f"[SCHEDULER] Consolidation job failed for '{subject_id}': {e}") 

event_filter.py

# event_filter.py """ Significance-based ingestion gate for Memoria Machinae. The EventFilter solves the memory saturation problem: a robot perceiving the world at 30fps would generate millions of near-identical memories per hour without filtering. This module scores each candidate event and only passes it to the ingestion pipeline if it clears the configured threshold. Default scoring strategy (override by subclassing EventFilter): - Base score: 0.1 (everything starts with a small chance of passing) - +0.5 if the text contains a configured significant keyword - +0.3 if an image is provided (visual novelty assumed) - +0.2 if text length is >= 80 characters (detail implies significance) - +0.1 if metadata contains 'priority': 'high' Capped at 1.0. Threshold is set via CONFIG['event_filter_threshold'] (default 0.3). Events scoring below threshold are dropped and counted in Prometheus. Usage: event_filter = EventFilter(subject_id='robot_001') if event_filter.should_ingest(text, image_path=path, metadata=meta): await stack.enqueue(text=text, image_path=path, metadata=meta) For custom scoring, subclass EventFilter and override score(): class MyFilter(EventFilter): def score(self, text, image_path=None, image_bytes=None, metadata=None): # your logic — e.g. pull vision novelty score from a separate model return 0.0–1.0 """ import logging from config import CONFIG from metrics import EVENT_FILTER_PASSED, EVENT_FILTER_DROPPED logger = logging.getLogger(__name__) class EventFilter: """ Scores candidate events and gates ingestion by significance threshold. Args: subject_id: The agent/robot subject this filter belongs to. threshold: Override the config threshold for this instance. """ def __init__(self, subject_id: str, threshold: float | None = None): self.subject_id = subject_id self.threshold = threshold if threshold is not None else CONFIG['event_filter_threshold'] self._keywords = CONFIG['event_significant_keywords'] def score( self, text: str, image_path: str | None = None, image_bytes: bytes | None = None, metadata: dict | None = None, ) -> float: """ Compute a significance score in [0.0, 1.0] for a candidate event. Override this method in a subclass to implement domain-specific scoring (e.g. object novelty from a vision model, task criticality from a mission planner, etc.). Args: text: Text description of the event. image_path: Path to an image associated with the event. image_bytes: Raw image bytes (alternative to image_path). metadata: Optional dict of additional context. Returns: Score in [0.0, 1.0]. Higher = more significant. """ s = 0.1 # base — everything has a floor chance text_lower = text.lower() if any(kw in text_lower for kw in self._keywords): s += 0.5 if image_path or image_bytes: s += 0.3 if len(text) >= 80: s += 0.2 if metadata and metadata.get('priority') == 'high': s += 0.1 return min(1.0, s) def should_ingest( self, text: str, image_path: str | None = None, image_bytes: bytes | None = None, metadata: dict | None = None, ) -> bool: """ Returns True if the event should be ingested, False if it should be dropped. Logs at DEBUG level for passing events and DEBUG level for dropped events, and updates Prometheus counters. """ significance = self.score(text, image_path, image_bytes, metadata) if significance >= self.threshold: EVENT_FILTER_PASSED.labels(subject_id=self.subject_id).inc() logger.debug( f"[FILTER:{self.subject_id}] PASS score={significance:.2f} " f"threshold={self.threshold:.2f} text='{text[:60]}...'" ) return True EVENT_FILTER_DROPPED.labels( subject_id=self.subject_id, reason='below_threshold' ).inc() logger.debug( f"[FILTER:{self.subject_id}] DROP score={significance:.2f} " f"threshold={self.threshold:.2f} text='{text[:60]}...'" ) return False 

feedback.py

# feedback.py """ Structured action-outcome feedback ingestion for Memoria Machinae. This module closes the learning loop: the pattern it implements: 1. Robot plans and executes an action. 2. Robot observes the outcome (success/failure, sensor readings, errors). 3. FeedbackIngester formats a structured memory that links action → outcome and enqueues it for ingestion. Without this loop, the memory graph accumulates observations but never builds the causal chains that make relational recall useful (e.g. "when surface is wet AND gripper is cold, grasp force must be +20%"). Usage: from feedback import FeedbackIngester, ActionOutcome ingester = FeedbackIngester(stack) outcome = ActionOutcome( action="grasp red cube on shelf B", success=False, context={"surface": "wet", "gripper_temp_c": 8.2}, sensor_readings={"grip_force_n": 14.5, "slip_detected": True}, error="grip slip detected at 14.5N", correction="increase grip force to 22N", ) await ingester.record(outcome) The resulting memory text is: "ACTION: grasp red cube on shelf B | OUTCOME: FAILED | ERROR: grip slip detected at 14.5N | CONTEXT: surface=wet gripper_temp_c=8.2 | SENSORS: grip_force_n=14.5 slip_detected=True | CORRECTION: increase grip force to 22N" This structured format ensures the entity extractor (spaCy) and graph edges capture the right relationships, and the keyword scorer in EventFilter assigns it a high significance score automatically. """ import logging from dataclasses import dataclass, field logger = logging.getLogger(__name__) @dataclass class ActionOutcome: """ Structured representation of a robot action and its observed outcome. Args: action: Natural-language description of what the robot tried to do. success: Whether the action succeeded. context: Environmental context at the time of action (dict). sensor_readings: Relevant sensor values at the time of outcome (dict). error: Error description if the action failed. correction: What should be done differently next time. image_path: Optional path to an image captured at outcome time. image_bytes: Optional raw image bytes (alternative to image_path). metadata: Additional metadata passed through to the memory node. """ action: str success: bool context: dict = field(default_factory=dict) sensor_readings: dict = field(default_factory=dict) error: str | None = None correction: str | None = None image_path: str | None = None image_bytes: bytes | None = None metadata: dict = field(default_factory=dict) def to_text(self) -> str: """ Render the outcome as a structured text string suitable for ingestion. The format is fixed and keyword-rich to ensure: - EventFilter scores it above threshold (contains outcome keywords). - spaCy entity extraction captures relevant noun phrases. - Graph edges link this outcome to future similar situations. """ parts = [ f"ACTION: {self.action}", f"OUTCOME: {'SUCCEEDED' if self.success else 'FAILED'}", ] if self.error: parts.append(f"ERROR: {self.error}") if self.context: ctx_str = ' '.join(f"{k}={v}" for k, v in self.context.items()) parts.append(f"CONTEXT: {ctx_str}") if self.sensor_readings: sensor_str = ' '.join(f"{k}={v}" for k, v in self.sensor_readings.items()) parts.append(f"SENSORS: {sensor_str}") if self.correction: parts.append(f"CORRECTION: {self.correction}") return ' | '.join(parts) class FeedbackIngester: """ Wraps a MemoriaStack to provide structured action-outcome feedback ingestion. This is a convenience class — you could call stack.enqueue() directly with outcome.to_text(). FeedbackIngester adds logging, priority metadata tagging, and a clean interface that separates feedback ingestion from general ingestion in calling code. Args: stack: A MemoriaStack instance for the agent/robot subject. """ def __init__(self, stack): self._stack = stack self.subject_id = stack.subject_id async def record(self, outcome: ActionOutcome) -> bool: """ Ingest a structured action outcome into memory. Failures are tagged 'priority: high' so EventFilter scores them above threshold regardless of other signals. Args: outcome: ActionOutcome dataclass. Returns: True if successfully enqueued, False if queue is full. """ text = outcome.to_text() metadata = { **outcome.metadata, 'feedback': True, 'success': outcome.success, } if not outcome.success: metadata['priority'] = 'high' accepted = await self._stack.enqueue( text=text, image_path=outcome.image_path, image_bytes=outcome.image_bytes, metadata=metadata, ) if accepted: logger.info( f"[FEEDBACK:{self.subject_id}] Recorded outcome — " f"action='{outcome.action[:60]}' success={outcome.success}" ) else: logger.warning( f"[FEEDBACK:{self.subject_id}] Queue full — outcome dropped for " f"action='{outcome.action[:60]}'" ) return accepted

# query_builder.py """ Recall query templates for robot planning contexts. Raw string queries to the recall engine are expressive but inconsistent across callers: different parts of a robot's control stack may describe the same planning need in very different words, producing inconsistent recall results. RecallQueryBuilder provides a library of structured query templates covering the most common robot planning contexts. Each template is designed to surface the most relevant class of memories for that context. Usage: from query_builder import RecallQueryBuilder qb = RecallQueryBuilder() # Before a manipulation task query, weights = qb.manipulation(object_description="wet red cube") results = await stack.recall(query=query, weights=weights) # After entering a new room query, weights = qb.environment_change(observation="dark corridor, cables on floor") results = await stack.recall(query=query, weights=weights) # When the same task has failed before query, weights = qb.failure_recovery(task="pick and place", error="grip slip") results = await stack.recall(query=query, weights=weights) Each method returns a (query_string, RecallWeights) tuple so callers get appropriate blend weights alongside the query without needing to know which weight profile fits which context. # models/manager.py """ Model manager — lazy-loaded singleton for all ML models. v1.1 changes: - CLIP (openai/clip-vit-base-patch32) replaced with SigLIP (google/siglip-base-patch16-224). SigLIP uses a sigmoid loss instead of softmax, giving better calibrated vision-language scores, especially for longer captions. """ import json import logging import sqlite3 import time from config import CONFIG, ENCRYPTION_KEY logger = logging.getLogger(__name__) _POSTGRES_DSN = CONFIG.get('postgres_dsn', '') class BlackBox: def __init__(self, subject_id: str): self.subject_id = subject_id self._pool = None self._sqlite_conn = None if not _POSTGRES_DSN: self._sqlite_conn = sqlite3.connect(CONFIG['sqlite_blackbox_path']) self._sqlite_conn.execute(""" CREATE TABLE IF NOT EXISTS audit_log ( id INTEGER PRIMARY KEY AUTOINCREMENT, subject_id TEXT NOT NULL, operation TEXT NOT NULL, memory_id TEXT NOT NULL, timestamp REAL NOT NULL, payload BLOB NOT NULL ) """) self._sqlite_conn.execute( "CREATE INDEX IF NOT EXISTS idx_subject ON audit_log (subject_id)" ) self._sqlite_conn.execute( "CREATE INDEX IF NOT EXISTS idx_operation ON audit_log (operation)" ) self._sqlite_conn.execute( "CREATE INDEX IF NOT EXISTS idx_timestamp ON audit_log (timestamp)" ) self._sqlite_conn.commit() logger.warning("[BLACKBOX] Using SQLite (dev mode). """ clauses = [] params: list = [] if subject_id: clauses.append("subject_id = ?") params.append(subject_id) if operation: clauses.append("operation = ?") params.append(operation) if memory_id: clauses.append("memory_id = ?") params.append(memory_id) if since is not None: clauses.append("timestamp >= ?") params.append(since) if until is not None: clauses.append("timestamp <= ?") params.append(until) where = ("WHERE " + " AND ".join(clauses)) if clauses else "" sql = f"SELECT payload FROM audit_log {where} ORDER BY timestamp ASC LIMIT ?" params.append(limit) conn = sqlite3.connect(self._db_path) try: rows = conn.execute(sql, params).fetchall() finally: conn.close() results = [] for (payload,) in rows: try: results.append(self._decrypt(payload)) except Exception as e: logger.warning(f"[BLACKBOX_QUERY] Failed to decrypt entry: {e}") return results def summary(self, subject_id: str | None = None) -> dict: """ Return a high-level summary of operations in the audit log. # recall/recall.py """ Fused recall engine with configurable per-call RecallWeights. Cold-start note: The graph component of the fused score is meaningless until the graph has sufficient density (~500+ nodes with diverse entity overlap). A warning is logged on each recall when active node count is below 50. This is expected behaviour during early deployment. """ import logging import time from dataclasses import dataclass, field import numpy as np from config import CONFIG from memory.graph import EpisodicNode, MemoryGraph from memory.store import VectorStore from metrics import RECALL_LATENCY, MEMORY_RECALLED_COUNT from models.multimodal_fusion import generate_multimodal_embedding logger = logging.getLogger(__name__) _COLD_START_THRESHOLD = 50 @dataclass class RecallWeights: """ Blend weights for the fused recall score. Tuning guide: Long-horizon agents: increase strength, decrease recency Conversational agents: increase recency Causal / relational: increase graph Pure semantic RAG: set vector high, others near zero """ vector: float = field(default_factory=lambda: CONFIG['recall_default_weights']['vector']) graph: float = field(default_factory=lambda: CONFIG['recall_default_weights']['graph']) strength: float = field(default_factory=lambda: CONFIG['recall_default_weights']['strength']) recency: float = field(default_factory=lambda: CONFIG['recall_default_weights']['recency']) def validate(self) -> None: total = self.vector + self.graph + self.strength + self.recency if abs(total - 1.0) > 0.01: logger.warning( f"RecallWeights sum to {total:.3f} (expected 1.0). " "Scores will be unnormalised." ) class RecallEngine: def __init__(self, subject_id: str, graph: MemoryGraph, store: VectorStore): self.subject_id = subject_id self.graph = graph self.store = store async def recall( self, query: str, k: int | None = None, min_strength: float | None = None, weights: RecallWeights | None = None, image_path: str | None = None, image_bytes: bytes | None = None, ) -> list[EpisodicNode]: t_start = time.time() k = k or CONFIG['recall_default_k'] min_strength = min_strength if min_strength is not None else CONFIG['recall_default_min_strength'] w = weights or RecallWeights() w.validate() # Cold-start warning active_count = len(self.graph.all_nodes()) if active_count < _COLD_START_THRESHOLD: logger.warning( f"[RECALL:{self.subject_id}] Cold start — only {active_count} active nodes. " f"Graph-based recall will be weak until >{_COLD_START_THRESHOLD} nodes exist. " "This is expected during early deployment." ) # 1. Query embedding query_tensor, _ = await generate_multimodal_embedding( query, image_path=image_path, image_bytes=image_bytes ) query_np = query_tensor.squeeze(0).detach().numpy() # 2. Vector search candidate_ids, vector_scores = self.store.search(query_np, k=k * 3) vector_score_map = dict(zip(candidate_ids, vector_scores)) # 3. Graph neighbourhood expansion graph_boost: dict[str, float] = {} for mid in candidate_ids[:k]: for neighbour in self.graph.get_neighbours(mid): if neighbour.memory_id not in vector_score_map: graph_boost[neighbour.memory_id] = ( graph_boost.get(neighbour.memory_id, 0.0) + 0.1 ) all_ids = list(set(candidate_ids) | set(graph_boost.keys())) # 4. Fused scoring scored: list[tuple[float, EpisodicNode]] = [] for mid in all_ids: node = self.graph.get_node(mid) if node is None or node.deleted or node.strength < min_strength: continue fused = ( w.vector * vector_score_map.get(mid, 0.0) + w.graph * min(1.0, graph_boost.get(mid, 0.0)) + w.strength * node.strength + w.recency * node.recency_score() ) scored.append((fused, node)) # 5. Deduplicate by cosine diversity scored.sort(key=lambda x: x[0], reverse=True) results: list[EpisodicNode] = [] selected_embeddings: list[np.ndarray] = [] threshold = CONFIG['diversity_threshold'] for _, node in scored: emb = node.embedding if selected_embeddings: sims = [ float(np.dot(emb, s) / (np.linalg.norm(emb) * np.linalg.norm(s) + 1e-8)) for s in selected_embeddings ] if max(sims) >= threshold: continue results.append(node) selected_embeddings.append(emb) node.touch() if len(results) >= k: break RECALL_LATENCY.observe(time.time() - t_start) MEMORY_RECALLED_COUNT.labels(subject_id=self.subject_id).inc() return results

""" import logging from config import CONFIG from ethics.blackbox import BlackBox from ethics.sentience import SentienceGovernor from memory.graph import MemoryGraph from memory.store import VectorStore from pipeline.pipeline import MemoryPipeline, IngestionRequest from recall.recall import RecallEngine, RecallWeights from metrics import CONSOLIDATION_COUNT, NODES_MERGED_COUNT, NODES_PRUNED_COUNT logger = logging.getLogger(__name__) class MemoriaStack: """ Governs the full memory lifecycle for a single subject_id. ── Consolidation ───────────────────────────────────────────────────────── async def consolidate(self) -> dict: import numpy as np merged_count = 0 pruned_count = 0 synced_count = 0 active = self.graph.all_nodes() threshold = CONFIG['consolidation_merge_threshold'] visited: set[str] = set() for i, node_a in enumerate(active): if node_a.memory_id in visited: continue for node_b in active[i + 1:]: if node_b.memory_id in visited: continue emb_a = node_a.embedding emb_b = node_b.embedding norm_a = emb_a / (emb_a @ emb_a) ** 0.5 norm_b = emb_b / (emb_b @ emb_b) ** 0.5 sim = float(norm_a @ norm_b) if sim >= threshold: if self.graph.merge_nodes(node_a.memory_id, node_b.memory_id): self.store.update_embedding(node_a.memory_id, node_a.embedding) self.store.remove(node_b.memory_id) visited.add(node_b.memory_id) merged_count += 1 await self.blackbox.log('merge', node_a.memory_id, { 'secondary': node_b.memory_id, 'similarity': sim, }) pruned_ids = self.graph.prune_weak_nodes() pruned_count = len(pruned_ids) for mid in pruned_ids: await self.blackbox.log('prune', mid) for node in self.graph.all_nodes(include_deleted=True): if node.deleted: if self.store.remove(node.memory_id): synced_count += 1 for _ in range(CONFIG['dream_replay_steps']): self.graph.strengthen_high_salience_edges() CONSOLIDATION_COUNT.labels(subject_id=self.subject_id).inc() NODES_MERGED_COUNT.labels(subject_id=self.subject_id).inc(merged_count) NODES_PRUNED_COUNT.labels(subject_id=self.subject_id).inc(pruned_count) stats = { 'subject_id': self.subject_id, 'merged': merged_count, 'pruned': pruned_count, 'synced_deletes': synced_count, } logger.info(f"[STACK:{self.subject_id}] Consolidation complete: {stats}") return stats # Design Notes

What Changed in v1.1

SigLIP replaces CLIP. google/siglip-base-patch16-224 uses a sigmoid loss over softmax, which produces better-calibrated confidence for individual image-text pairs (rather than treating them as a closed set). In practice this improves retrieval quality when the description and image are loosely paired, which is common during robot perception. The IMAGE_DIM is now 768 (up from 512); the CrossModalFusion projection layer absorbs this difference transparently.

Transformer-based NER with a robotics ruler. Upgrading from en_core_web_sm (CNN-based) to en_core_web_trf (transformer-based) gives substantially better entity recall on technical language. The layered EntityRuler handles domain-specific vocabulary — object classes (cube, gripper), action verbs (grasp, stack), sensor names (lidar, IMU), and location labels (zone A, dock) — that general-purpose NER consistently misses. Extend CONFIG['robotics_entity_vocab'] to cover your specific payload.

BlackBoxQueryEngine. Forensic querying was previously only possible via direct SQL. The new BlackBoxQueryEngine provides a decrypting Python interface: filter by subject, operation, memory ID, and time range. Useful for post-incident review and compliance audits. PostgreSQL support is flagged as a future TODO — the decryption key and schema are documented for teams who need to query PostgreSQL directly.

Test suite. Four test modules covering the modules with the most non-trivial logic: EventFilter (scoring, threshold gating, subclassing), FeedbackIngester (text format, priority tagging, queue-full handling), RecallEngine (empty store, strength filtering, cold-start warning, weight validation, diversity filter), and consolidation (merge, prune, sync, audit logging). All ML model calls are patched out via conftest.py so tests run on CI without GPU.

What Has Not Changed

All core subsystems are preserved intact: HNSW/IVFPQ vector store, episodic graph with NetworkX, SNN pipeline with LIF neurons, async ingestion queue, APScheduler background maintenance, PostgreSQL + Fernet audit ledger, FastAPI service, JWT auth, Prometheus metrics, EventFilter, FeedbackIngester, and RecallQueryBuilder.

Cold Start

The graph component of fused recall is meaningless until approximately 500+ nodes with diverse entity overlap exist. Expect the first hours of deployment to behave essentially as vector-only semantic search. The RecallEngine logs a warning when active node count is below 50.

Embedding latency (150–400ms on Jetson) means the memory service must never block the real-time control loop. Use microservice mode and enqueue() (fire-and-forget) for perception ingestion. Use acquire() (blocking) only for high-priority memories where you need the node ID immediately.

Note: en_core_web_trf requires more RAM than en_core_web_sm (~400MB vs ~15MB). On Raspberry Pi 5 or other constrained devices, override with SPACY_MODEL=en_core_web_sm to use the lightweight model at the cost of NER accuracy.
