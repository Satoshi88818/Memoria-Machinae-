Memoria Machinae v1.2

Production-grade episodic memory microservice for embodied and language agents.

Changes in v1.2 over v1.1

Explicit causal edges on ActionOutcome ingestion — feedback.py now passes a causal=True flag through metadata; pipeline/pipeline.py detects this and calls graph.add_causal_edge(action_node_id, outcome_node_id) after storing both nodes. memory/graph.py gains add_causal_edge().

Schema induction during consolidation — stack.py consolidation pass now calls SchemaInducer.detect() after merging. Repeated action patterns (≥3 nodes sharing the same ACTION entity + outcome label) are promoted to named schema nodes and connected with schema_of edges in the graph.

Temporal window queries in recall — RecallEngine.recall() accepts optional since / until Unix timestamps; nodes outside the window are excluded before scoring. RecallQueryBuilder gains a temporal_window() helper.

Lightweight salience detector — models/salience.py adds EmbeddingStatsSalienceDetector that scores novelty purely from embedding statistics (mean cosine distance to recent embeddings + entity novelty) without a full SNN pass. Activated when USE_SNN=false.

Memory type support — EpisodicNode gains a memory_type field (episodic / semantic / procedural). Each type has distinct decay and consolidation rules in graph.py and stack.py.

Retrieval-time cross-encoder re-ranking — recall/recall.py applies a small cross-encoder (cross-encoder/ms-marco-MiniLM-L-6-v2) to re-rank the top-k candidates after fused scoring. Controlled by CONFIG['use_cross_encoder'].

# models/salience.py """ Lightweight salience detector for edge devices (USE_SNN=false path). Instead of a full Spiking Neural Network, this module computes novelty from two cheap embedding statistics: 1. Mean cosine distance to a rolling window of recent embeddings. High distance → the current perception is different from recent history. 2. Entity novelty: fraction of extracted entities not seen in recent memories. A composite score is returned in [0.0, 1.0]. Nodes scoring above CONFIG['salience_novelty_threshold'] are marked high_salience. Usage: detector = EmbeddingStatsSalienceDetector(subject_id='robot_001') score, is_high_salience = detector.score(embedding_np, entities=['cube', 'shelf']) detector.update(embedding_np, entities=['cube', 'shelf']) The detector is NOT thread-safe. It is per-subject and per-pipeline instance. """ import logging from collections import deque import numpy as np from config import CONFIG logger = logging.getLogger(__name__) class EmbeddingStatsSalienceDetector: """ Scores novelty from embedding cosine distance + entity frequency. Args: subject_id: Agent subject, used only for logging. window_size: Number of recent embeddings to keep (overrides config). threshold: Score above which high_salience is True (overrides config). """ def __init__( self, subject_id: str, window_size: int | None = None, threshold: float | None = None, ): self.subject_id = subject_id self._window_size = window_size or CONFIG['salience_novelty_window'] self._threshold = threshold or CONFIG['salience_novelty_threshold'] self._embedding_window: deque[np.ndarray] = deque(maxlen=self._window_size) self._entity_window: deque[set[str]] = deque(maxlen=self._window_size) # ── Scoring ────────────────────────────────────────────────────────────── def score( self, embedding: np.ndarray, entities: list[str] | None = None, ) -> tuple[float, bool]: """ Compute a novelty score for an incoming perception. Returns: (score: float in [0.0, 1.0], is_high_salience: bool) """ emb_score = self._embedding_novelty(embedding) entity_score = self._entity_novelty(entities or []) # Weighted composite: embedding distance carries more signal composite = min(1.0, 0.7 * emb_score + 0.3 * entity_score) high = composite >= self._threshold logger.debug( f"[SALIENCE:{self.subject_id}] " f"emb={emb_score:.3f} entity={entity_score:.3f} " f"composite={composite:.3f} high={high}" ) return composite, high def update(self, embedding: np.ndarray, entities: list[str] | None = None) -> None: """Add the current perception to the rolling window.""" norm = embedding / (np.linalg.norm(embedding) + 1e-8) self._embedding_window.append(norm) self._entity_window.append(set(entities or [])) # ── Internal helpers ───────────────────────────────────────────────────── def _embedding_novelty(self, embedding: np.ndarray) -> float: """Mean cosine distance to recent window. Empty window → 1.0 (fully novel).""" if not self._embedding_window: return 1.0 norm = embedding / (np.linalg.norm(embedding) + 1e-8) recent = np.stack(list(self._embedding_window)) # [W, D] sims = recent @ norm # [W] mean_sim = float(np.mean(sims)) return max(0.0, min(1.0, 1.0 - mean_sim)) def _entity_novelty(self, entities: list[str]) -> float: """Fraction of entities not seen in the recent entity window.""" if not entities: return 0.0 if not self._entity_window: return 1.0 seen: set[str] = set().union(*self._entity_window) novel = [e for e in entities if e not in seen] return len(novel) / len(entities)

Design Notes

What Changed in v1.2

1. Explicit causal edges.
ActionOutcome now carries an optional action_memory_id. When the FeedbackIngester records an outcome, it sets metadata['causal'] = True. The pipeline detects this flag and calls graph.add_causal_edge(action_id, outcome_id) after storing the outcome node. Causal edges are typed (edge_type='causal', weight=1.0) and are distinguishable from co-occurrence edges in all graph queries. The new get_causal_successors() method on MemoryGraph lets planners efficiently traverse the causal chain.

2. Schema induction.
schema/inducer.py introduces SchemaInducer, which runs as the final step of every consolidation pass. It groups episodic nodes by (ACTION_entity, OUTCOME_label) and promotes any group of ≥ schema_min_pattern_count nodes to a procedural schema node. Schema nodes are centroid embeddings of their source cluster and are connected to member nodes via schema_of edges. Repeated induction strengthens existing schema nodes rather than duplicating them. Schema nodes are immune to weak-node pruning.

3. Temporal window queries.
RecallEngine.recall() accepts since / until Unix timestamps. Nodes outside the window are excluded before fused scoring, not after — this avoids scoring overhead on irrelevant memories. RecallQueryBuilder.temporal_window() computes the window bounds from a topic string and a window_seconds duration, returning a ready-to-use 4-tuple.

4. Lightweight salience detector.
models/salience.py adds EmbeddingStatsSalienceDetector. When USE_SNN=false, the pipeline calls this instead of the SNN. It scores novelty from mean cosine distance to a rolling embedding window (70% weight) plus entity novelty relative to recent entity history (30% weight). This runs in microseconds on any hardware, making it suitable for Raspberry Pi 5 and similar edge devices. The SpikeProfile field is populated with a minimal stub for graph compatibility.

5. Memory types.
EpisodicNode gains memory_type: Literal['episodic', 'semantic', 'procedural']. Each type has its own decay rate in config (episodic: 0.001, semantic: 0.0002, procedural: 0.00005). Only episodic nodes are eligible for consolidation merging and pruning. The REST API exposes memory_type as an optional field on both acquire and recall endpoints.

6. Cross-encoder re-ranking.
After diversity-filtered candidates are selected, RecallEngine passes the top cross_encoder_rerank_top_n (default: 20) (query, node_text) pairs through cross-encoder/ms-marco-MiniLM-L-6-v2. This model scores pairs jointly (not independently), catching relevance signals that bi-encoder cosine similarity misses. Set USE_CROSS_ENCODER=false to disable for latency-critical deployments.

What Has Not Changed

All core subsystems from v1.1 are preserved: HNSW/IVFPQ vector store, episodic graph, SNN pipeline with LIF neurons, async ingestion queue, APScheduler background maintenance, PostgreSQL + Fernet audit ledger, FastAPI service, JWT auth, Prometheus metrics, EventFilter, FeedbackIngester structured text format, and RecallQueryBuilder templates.
