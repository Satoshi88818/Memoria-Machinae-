Memoria Machinae v1.0

Production-grade episodic memory microservice for embodied and language agents.

Memoria Machinae is a renamed and extended fork of SoulCapsule Elevated v3.2, purpose-built for robot and agent integration. It adds four capabilities not present in the original: an event filter layer that prevents memory saturation from indiscriminate ingestion, a feedback ingestion helper that closes the action-outcome loop required for relational learning, configurable recall query templates for structured planning contexts, and live camera / bytes input for the multimodal pipeline so robots don't need to write temp files on every frame.

All original subsystems (HNSW/IVFPQ vector store, episodic graph, optional SNN, async ingestion workers, background APScheduler maintenance, PostgreSQL audit ledger, FastAPI microservice, JWT auth) are preserved and unchanged except where noted.

# event_filter.py """ Significance-based ingestion gate for Memoria Machinae. The EventFilter solves the memory saturation problem: a robot perceiving the world at 30fps would generate millions of near-identical memories per hour without filtering. This module scores each candidate event and only passes it to the ingestion pipeline if it clears the configured threshold. Default scoring strategy (override by subclassing EventFilter): - Base score: 0.1 (everything starts with a small chance of passing) - +0.5 if the text contains a configured significant keyword - +0.3 if an image is provided (visual novelty assumed) - +0.2 if text length is >= 80 characters (detail implies significance) - +0.1 if metadata contains 'priority': 'high' Capped at 1.0. Threshold is set via CONFIG['event_filter_threshold'] (default 0.3). Events scoring below threshold are dropped and counted in Prometheus. Usage: event_filter = EventFilter(subject_id='robot_001') if event_filter.should_ingest(text, image_path=path, metadata=meta): await stack.enqueue(text=text, image_path=path, metadata=meta) For custom scoring, subclass EventFilter and override score(): class MyFilter(EventFilter): def score(self, text, image_path=None, image_bytes=None, metadata=None): # your logic return 0.0–1.0 """ import logging from config import CONFIG from metrics import EVENT_FILTER_PASSED, EVENT_FILTER_DROPPED logger = logging.getLogger(__name__) class EventFilter: """ Scores candidate events and gates ingestion by significance threshold. Args: subject_id: The agent/robot subject this filter belongs to. threshold: Override the config threshold for this instance. """ def __init__(self, subject_id: str, threshold: float | None = None): self.subject_id = subject_id self.threshold = threshold if threshold is not None else CONFIG['event_filter_threshold'] self._keywords = CONFIG['event_significant_keywords'] def score( self, text: str, image_path: str | None = None, image_bytes: bytes | None = None, metadata: dict | None = None, ) -> float: """ Compute a significance score in [0.0, 1.0] for a candidate event. Override this method in a subclass to implement domain-specific scoring (e.g. object novelty from a vision model, task criticality from a mission planner, etc.). Args: text: Text description of the event. image_path: Path to an image associated with the event. image_bytes: Raw image bytes (alternative to image_path). metadata: Optional dict of additional context. Returns: Score in [0.0, 1.0]. Higher = more significant. """ s = 0.1 # base — everything has a floor chance # Keyword match text_lower = text.lower() if any(kw in text_lower for kw in self._keywords): s += 0.5 # Visual novelty (image present = worth remembering) if image_path or image_bytes: s += 0.3 # Detail heuristic (longer descriptions tend to describe significant events) if len(text) >= 80: s += 0.2 # Explicit priority flag from upstream system if metadata and metadata.get('priority') == 'high': s += 0.1 return min(1.0, s) def should_ingest( self, text: str, image_path: str | None = None, image_bytes: bytes | None = None, metadata: dict | None = None, ) -> bool: """ Returns True if the event should be ingested, False if it should be dropped. Logs at DEBUG level for passing events and INFO level for dropped events, and updates Prometheus counters. """ significance = self.score(text, image_path, image_bytes, metadata) if significance >= self.threshold: EVENT_FILTER_PASSED.labels(subject_id=self.subject_id).inc() logger.debug( f"[FILTER:{self.subject_id}] PASS score={significance:.2f} " f"threshold={self.threshold:.2f} text='{text[:60]}...'" ) return True EVENT_FILTER_DROPPED.labels( subject_id=self.subject_id, reason='below_threshold' ).inc() logger.debug( f"[FILTER:{self.subject_id}] DROP score={significance:.2f} " f"threshold={self.threshold:.2f} text='{text[:60]}...'" ) return False

# feedback.py """ Structured action-outcome feedback ingestion for Memoria Machinae. This module closes the learning loop that the original SCE v3.2 codebase left open. The pattern it implements: 1. Robot plans and executes an action. 2. Robot observes the outcome (success/failure, sensor readings, errors). 3. FeedbackIngester formats a structured memory that links action → outcome and enqueues it for ingestion. Without this loop, the memory graph accumulates observations but never builds the causal chains that make relational recall useful (e.g. "when surface is wet AND gripper is cold, grasp force must be +20%"). Usage: from feedback import FeedbackIngester, ActionOutcome ingester = FeedbackIngester(stack) outcome = ActionOutcome( action="grasp red cube on shelf B", success=False, context={"surface": "wet", "gripper_temp_c": 8.2}, sensor_readings={"grip_force_n": 14.5, "slip_detected": True}, error="grip slip detected at 14.5N", correction="increase grip force to 22N", ) await ingester.record(outcome) The resulting memory text is: "ACTION: grasp red cube on shelf B | OUTCOME: FAILED | ERROR: grip slip detected at 14.5N | CONTEXT: surface=wet gripper_temp_c=8.2 | SENSORS: grip_force_n=14.5 slip_detected=True | CORRECTION: increase grip force to 22N" This structured format ensures the entity extractor (spaCy) and graph edges capture the right relationships, and the keyword scorer in EventFilter assigns it a high significance score automatically. """ import logging from dataclasses import dataclass, field logger = logging.getLogger(__name__) @dataclass class ActionOutcome: """ Structured representation of a robot action and its observed outcome. Args: action: Natural-language description of what the robot tried to do. success: Whether the action succeeded. context: Environmental context at the time of action (dict). sensor_readings: Relevant sensor values at the time of outcome (dict). error: Error description if the action failed. correction: What should be done differently next time. image_path: Optional path to an image captured at outcome time. image_bytes: Optional raw image bytes (alternative to image_path). metadata: Additional metadata passed through to the memory node. """ action: str success: bool context: dict = field(default_factory=dict) sensor_readings: dict = field(default_factory=dict) error: str | None = None correction: str | None = None image_path: str | None = None image_bytes: bytes | None = None metadata: dict = field(default_factory=dict) def to_text(self) -> str: """ Render the outcome as a structured text string suitable for ingestion. The format is fixed and keyword-rich to ensure: - EventFilter scores it above threshold (contains outcome keywords). - spaCy entity extraction captures relevant noun phrases. - Graph edges link this outcome to future similar situations. """ parts = [ f"ACTION: {self.action}", f"OUTCOME: {'SUCCEEDED' if self.success else 'FAILED'}", ] if self.error: parts.append(f"ERROR: {self.error}") if self.context: ctx_str = ' '.join(f"{k}={v}" for k, v in self.context.items()) parts.append(f"CONTEXT: {ctx_str}") if self.sensor_readings: sensor_str = ' '.join(f"{k}={v}" for k, v in self.sensor_readings.items()) parts.append(f"SENSORS: {sensor_str}") if self.correction: parts.append(f"CORRECTION: {self.correction}") return ' | '.join(parts) class FeedbackIngester: """ Wraps a MemoriaStack to provide structured action-outcome feedback ingestion. This is a convenience class — you could call stack.enqueue() directly with outcome.to_text(). FeedbackIngester adds logging, priority metadata tagging, and a clean interface that separates feedback ingestion from general ingestion in calling code. Args: stack: A MemoriaStack instance for the agent/robot subject. """ def __init__(self, stack): self._stack = stack self.subject_id = stack.subject_id async def record(self, outcome: ActionOutcome) -> bool: """ Ingest a structured action outcome into memory. Failures are tagged 'priority: high' so EventFilter scores them above threshold regardless of other signals. Args: outcome: ActionOutcome dataclass. Returns: True if successfully enqueued, False if queue is full. """ text = outcome.to_text() metadata = { **outcome.metadata, 'feedback': True, 'success': outcome.success, } if not outcome.success: metadata['priority'] = 'high' accepted = await self._stack.enqueue( text=text, image_path=outcome.image_path, image_bytes=outcome.image_bytes, metadata=metadata, ) if accepted: logger.info( f"[FEEDBACK:{self.subject_id}] Recorded outcome — " f"action='{outcome.action[:60]}' success={outcome.success}" ) else: logger.warning( f"[FEEDBACK:{self.subject_id}] Queue full — outcome dropped for " f"action='{outcome.action[:60]}'" ) return accepted

# query_builder.py """ Recall query templates for robot planning contexts. Raw string queries to the recall engine are expressive but inconsistent across callers: different parts of a robot's control stack may describe the same planning need in very different words, producing inconsistent recall results. RecallQueryBuilder provides a library of structured query templates covering the most common robot planning contexts. Each template is designed to surface the most relevant class of memories for that context. Usage: from query_builder import RecallQueryBuilder from recall.recall import RecallWeights qb = RecallQueryBuilder() # Before a manipulation task query, weights = qb.manipulation(object_description="wet red cube") results = await stack.recall(query=query, weights=weights) # After entering a new room query, weights = qb.environment_change(observation="dark corridor, cables on floor") results = await stack.recall(query=query, weights=weights) # When the same task has failed before query, weights = qb.failure_recovery(task="pick and place", error="grip slip") results = await stack.recall(query=query, weights=weights) Each method returns a (query_string, RecallWeights) tuple so callers get appropriate blend weights alongside the query without needing to know which weight profile fits which context. """ import logging from recall.recall import RecallWeights from config import CONFIG logger = logging.getLogger(__name__) # Pre-defined weight profiles for different planning contexts. # These are starting points — tune for your robot's task distribution. _WEIGHTS_CAUSAL = RecallWeights( vector=0.30, graph=0.40, strength=0.20, recency=0.10 ) """Causal reasoning: strong graph bias surfaces action→outcome chains.""" _WEIGHTS_SEMANTIC = RecallWeights( vector=0.55, graph=0.15, strength=0.20, recency=0.10 ) """Semantic search: mirrors SCE v3.2 defaults. Good for object/concept lookup.""" _WEIGHTS_RECENCY = RecallWeights( vector=0.35, graph=0.20, strength=0.15, recency=0.30 ) """Recency-biased: prioritises fresh observations of a changing environment.""" _WEIGHTS_PROVEN = RecallWeights( vector=0.25, graph=0.35, strength=0.35, recency=0.05 ) """Proven strategies: strength-biased, surfaces consistently successful past actions.""" class RecallQueryBuilder: """ Provides structured recall query templates for common robot planning contexts. All methods return a (query: str, weights: RecallWeights) tuple. """ def manipulation( self, object_description: str, context: str | None = None, ) -> tuple[str, RecallWeights]: """ Query for past manipulation experience with a similar object or context. Returns causal weights to surface outcome chains (grip → succeed/fail). """ query = f"how did I handle grasping or manipulating {object_description}" if context: query += f" in conditions like {context}" logger.debug(f"[QUERY_BUILDER] manipulation: '{query}'") return query, _WEIGHTS_CAUSAL def environment_change( self, observation: str, ) -> tuple[str, RecallWeights]: """ Query for past experience in a similar environment. Returns recency weights because environment states change and stale memories of a space may be misleading. """ query = f"what did I observe or encounter in an environment like {observation}" logger.debug(f"[QUERY_BUILDER] environment_change: '{query}'") return query, _WEIGHTS_RECENCY def failure_recovery( self, task: str, error: str | None = None, ) -> tuple[str, RecallWeights]: """ Query for past failures on the same task and how they were resolved. Returns causal weights — we need to surface correction chains, not just surface similarity. """ query = f"how did I recover from failure while doing {task}" if error: query += f", specifically involving {error}" logger.debug(f"[QUERY_BUILDER] failure_recovery: '{query}'") return query, _WEIGHTS_CAUSAL def navigation( self, destination: str, current_context: str | None = None, ) -> tuple[str, RecallWeights]: """ Query for past navigation experience toward a similar destination. Returns recency weights because layout and obstacles change over time. """ query = f"what obstacles or routes did I encounter navigating to {destination}" if current_context: query += f" from {current_context}" logger.debug(f"[QUERY_BUILDER] navigation: '{query}'") return query, _WEIGHTS_RECENCY def operator_instruction( self, instruction_fragment: str, ) -> tuple[str, RecallWeights]: """ Query for past operator instructions similar to the current one. Returns semantic weights — we want close meaning match, not causal chains. """ query = f"operator instructions similar to: {instruction_fragment}" logger.debug(f"[QUERY_BUILDER] operator_instruction: '{query}'") return query, _WEIGHTS_SEMANTIC def proven_strategy( self, task: str, ) -> tuple[str, RecallWeights]: """ Query for the most reliably successful strategy for a task type. Returns proven-strategy weights — strength-biased to surface memories that have been reinforced by repeated success. """ query = f"what strategy succeeded most consistently for {task}" logger.debug(f"[QUERY_BUILDER] proven_strategy: '{query}'") return query, _WEIGHTS_PROVEN def novel_object( self, description: str, image_path: str | None = None, image_bytes: bytes | None = None, ) -> tuple[str, RecallWeights, str | None, bytes | None]: """ Query for past experience with visually or semantically similar objects. Optionally includes an image for multimodal recall. Returns a 4-tuple: (query, weights, image_path, image_bytes). """ query = f"what do I know about objects similar to {description}" logger.debug(f"[QUERY_BUILDER] novel_object: '{query}'") return query, _WEIGHTS_SEMANTIC, image_path, image_bytes

# recall/recall.py # Unchanged from SCE v3.2 except logging a cold-start warning when node # count is low (< 50 active nodes) to surface the cold-start period clearly. """ Fused recall engine with configurable per-call RecallWeights. Cold-start note: The graph component of the fused score is meaningless until the graph has sufficient density (~500+ nodes with diverse entity overlap). A warning is logged on each recall when active node count is below 50. This is expected behaviour during early deployment. """ import logging import time from dataclasses import dataclass, field import numpy as np from config import CONFIG from memory.graph import EpisodicNode, MemoryGraph from memory.store import VectorStore from metrics import RECALL_LATENCY, MEMORY_RECALLED_COUNT from models.multimodal_fusion import generate_multimodal_embedding logger = logging.getLogger(__name__) _COLD_START_THRESHOLD = 50 @dataclass class RecallWeights: """ Blend weights for the fused recall score. Tuning guide: Long-horizon agents: increase strength, decrease recency Conversational agents: increase recency Causal / relational: increase graph Pure semantic RAG: set vector high, others near zero """ vector: float = field(default_factory=lambda: CONFIG['recall_default_weights']['vector']) graph: float = field(default_factory=lambda: CONFIG['recall_default_weights']['graph']) strength: float = field(default_factory=lambda: CONFIG['recall_default_weights']['strength']) recency: float = field(default_factory=lambda: CONFIG['recall_default_weights']['recency']) def validate(self) -> None: total = self.vector + self.graph + self.strength + self.recency if abs(total - 1.0) > 0.01: logger.warning( f"RecallWeights sum to {total:.3f} (expected 1.0). Scores will be unnormalised." ) class RecallEngine: def __init__(self, subject_id: str, graph: MemoryGraph, store: VectorStore): self.subject_id = subject_id self.graph = graph self.store = store async def recall( self, query: str, k: int | None = None, min_strength: float | None = None, weights: RecallWeights | None = None, image_path: str | None = None, image_bytes: bytes | None = None, ) -> list[EpisodicNode]: t_start = time.time() k = k or CONFIG['recall_default_k'] min_strength = min_strength if min_strength is not None else CONFIG['recall_default_min_strength'] w = weights or RecallWeights() w.validate() # Cold-start warning active_count = len(self.graph.all_nodes()) if active_count < _COLD_START_THRESHOLD: logger.warning( f"[RECALL:{self.subject_id}] Cold start — only {active_count} active nodes. " f"Graph-based recall will be weak until >{_COLD_START_THRESHOLD} nodes exist. " "This is expected during early deployment." ) # 1. Query embedding query_tensor, _ = await generate_multimodal_embedding( query, image_path=image_path, image_bytes=image_bytes ) query_np = query_tensor.squeeze(0).detach().numpy() # 2. Vector search candidate_ids, vector_scores = self.store.search(query_np, k=k * 3) vector_score_map = dict(zip(candidate_ids, vector_scores)) # 3. Graph neighbourhood expansion graph_boost: dict[str, float] = {} for mid in candidate_ids[:k]: for neighbour in self.graph.get_neighbours(mid): if neighbour.memory_id not in vector_score_map: graph_boost[neighbour.memory_id] = graph_boost.get(neighbour.memory_id, 0.0) + 0.1 all_ids = list(set(candidate_ids) | set(graph_boost.keys())) # 4. Fused scoring scored: list[tuple[float, EpisodicNode]] = [] for mid in all_ids: node = self.graph.get_node(mid) if node is None or node.deleted or node.strength < min_strength: continue fused = ( w.vector * vector_score_map.get(mid, 0.0) + w.graph * min(1.0, graph_boost.get(mid, 0.0)) + w.strength * node.strength + w.recency * node.recency_score() ) scored.append((fused, node)) # 5. Deduplicate by cosine diversity scored.sort(key=lambda x: x[0], reverse=True) results: list[EpisodicNode] = [] selected_embeddings: list[np.ndarray] = [] threshold = CONFIG['diversity_threshold'] for _, node in scored: emb = node.embedding if selected_embeddings: sims = [ float(np.dot(emb, s) / (np.linalg.norm(emb) * np.linalg.norm(s) + 1e-8)) for s in selected_embeddings ] if max(sims) >= threshold: continue results.append(node) selected_embeddings.append(emb) node.touch() if len(results) >= k: break RECALL_LATENCY.observe(time.time() - t_start) MEMORY_RECALLED_COUNT.labels(subject_id=self.subject_id).inc() return results

Design Notes

What Memoria Machinae Adds Over SCE v3.2

EventFilter solves the memory saturation problem that SCE v3.2 leaves entirely to the caller. Without it, a robot naively calling enqueue() on every perception cycle would generate millions of near-identical nodes per hour, destroying the signal-to-noise ratio in the graph and making consolidated recall useless. The filter is intentionally overridable by subclassing: the default scoring heuristic is reasonable for general robotics but any domain-specific signal (vision-based novelty scores, mission-planner criticality flags) can be injected cleanly.

FeedbackIngester closes the loop that SCE v3.2 describes but does not implement. The graph's relational reasoning only becomes valuable after thousands of action→outcome pairs are linked. The structured text format produced by ActionOutcome.to_text() is deliberately keyword-rich so the EventFilter passes it at high significance, and the entity extractor (spaCy) picks up the right noun phrases for graph edge formation.

RecallQueryBuilder makes recall consistent across a robot's subsystems. Without it, different components of a control stack (navigation, manipulation, high-level planning) will phrase the same planning need in different ways, producing inconsistent recall. The builder also packages the right RecallWeights with each query type so callers don't need to understand the weight space.

image_bytes support eliminates the temp-file pattern that SCE v3.2 requires for live camera input. Writing a temp file per frame at 30fps on a Jetson is both slow and a source of filesystem contention. Passing bytes directly removes disk I/O from the perception-memory path entirely.

What Has Not Changed

All core subsystems from SCE v3.2 are preserved intact: HNSW/IVFPQ vector store, episodic graph with NetworkX, SNN pipeline with LIF neurons, async ingestion queue, APScheduler background maintenance, PostgreSQL + Fernet audit ledger, FastAPI service, JWT auth, and Prometheus metrics. The architectural reasoning behind those choices remains valid and is documented in the SCE v3.2 design notes.

Cold Start

The graph component of fused recall is meaningless until approximately 500+ nodes with diverse entity overlap exist. Expect the first hours of deployment to behave essentially as vector-only semantic search. The RecallEngine now logs a warning when active node count is below 50 to make this period visible in logs rather than silently returning weak results.
