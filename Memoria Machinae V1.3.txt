Memoria Machinae v1.3

Production-grade episodic memory microservice for embodied and language agents.

Changes in v1.3 over v1.2

Hierarchical Schemas — schema/inducer.py now supports multi-level schema composition. A HierarchicalSchemaInducer detects when two or more procedural schema nodes share a common action sequence context and promotes them into a higher-order meta_schema node via composed_of edges. CONFIG gains schema_hierarchy_levels and schema_compose_min_count.

Precondition Learning — schema/precondition.py (NEW) traverses causal chains in the graph to extract the most common antecedent entity clusters for each schema node. Preconditions are stored as structured metadata on schema nodes and exposed via MemoryGraph.get_preconditions(). Background consolidation calls PreconditionLearner.extract() after schema induction.

Predictive Coding Integration — models/predictive.py (NEW) implements a lightweight forward model that predicts the next expected embedding given the current one. Prediction error (L2 distance between predicted and actual embedding) is used as an additional salience signal in both the SNN path (supplementing the spike profile) and the lightweight detector path. CONFIG gains use_predictive_coding, predictive_hidden_dim, and predictive_error_weight.

Multi-Agent / Social Memory — EpisodicNode gains a source_agent field and an observation_type field (self | observed | communicated). The MemoryGraph gains get_by_agent() and get_by_observation_type() query methods. RecallEngine gains an optional source_agent filter. FeedbackIngester passes source_agent through metadata. The REST API exposes source_agent and observation_type on acquire/recall endpoints.

Active Recall / Dream Phase — The consolidation dream_replay_steps loop is upgraded from a passive edge-strengthening pass to an active recall phase. stack.py's _dream_phase() selects the bottom-quartile causal nodes by strength that have high graph connectivity (≥ 2 causal successors) and synthetically re-queries the recall engine for their text, strengthening the nodes and their causal edges on each hit. Controlled by CONFIG['dream_active_recall'] (bool) and CONFIG['dream_replay_steps'].

Better Pattern Mining — schema/pattern_miner.py (NEW) replaces the naive entity-label grouping in SchemaInducer.detect() with two complementary algorithms: (a) graph community detection via the Louvain method (using networkx's greedy modularity communities) to identify structurally cohesive node clusters, and (b) sequential pattern mining (PrefixSpan-style) over temporally ordered entity sequences to find frequent action subsequences. Both feeds are merged before schema promotion.

# models/predictive.py """ Lightweight forward model for predictive coding integration (v1.3). The model learns to predict the next embedding given the current one via a simple two-layer MLP trained online with mean-squared error. Prediction error (L2 distance between predicted and actual embedding) is used as an additional salience signal: - High prediction error → the current perception is surprising → high salience. - Low prediction error → the current perception was anticipated → lower salience. """ def __init__( self, dim: int, hidden_dim: int = 256, subject_id: str = 'unknown', lr: float = 1e-3, device: str = 'cpu', ): self.subject_id = subject_id self.dim = dim self.device = torch.device(device) self._net = nn.Sequential( nn.Linear(dim, hidden_dim), nn.ReLU(), nn.Linear(hidden_dim, dim), ).to(self.device) self._optimizer = torch.optim.Adam(self._net.parameters(), lr=lr) self._loss_fn = nn.MSELoss() self._prev_emb: torch.Tensor | None = None self._step = 0 # ── Public API ─────────────────────────────────────────────────────────────── def predict_and_update(self, embedding: np.ndarray) -> tuple[float, np.ndarray]: """ Given the current embedding, compute prediction error against the previous prediction, update the model weights, and return the error. Returns: (prediction_error: float, predicted_next_embedding: np.ndarray) The first call always returns error=0.0 (no previous state to compare). # models/salience.py """ Lightweight salience detector for edge devices (USE_SNN=false path). v1.3: Integrates prediction error from the PredictiveCodingModel as a third salience signal when CONFIG['use_predictive_coding'] is True. Composite score = 0.7 * embedding_novelty + 0.3 * entity_novelty → when predictive coding enabled: (1 - predictive_error_weight) * above + predictive_error_weight * normalised_error """ import logging from collections import deque import numpy as np from config import CONFIG logger = logging.getLogger(__name__) class EmbeddingStatsSalienceDetector: """ Scores novelty from embedding cosine distance + entity frequency + optional prediction error signal. Args: subject_id: Agent subject, used only for logging. window_size: Number of recent embeddings to keep (overrides config). threshold: Score above which high_salience is True (overrides config). """ # Empirical upper bound for prediction RMSE used for normalisation. # Embeddings are unit-normed → max L2 between two vectors ≈ √2 ≈ 1.414. _PREDICTION_ERROR_SCALE = 1.414 def __init__( self, subject_id: str, window_size: int | None = None, threshold: float | None = None, ): self.subject_id = subject_id self._window_size = window_size or CONFIG['salience_novelty_window'] self._threshold = threshold or CONFIG['salience_novelty_threshold'] self._embedding_window: deque[np.ndarray] = deque(maxlen=self._window_size) self._entity_window: deque[set[str]] = deque(maxlen=self._window_size) # ── Scoring ────────────────────────────────────────────────────────────────── def score( self, embedding: np.ndarray, entities: list[str] | None = None, prediction_error: float | None = None, ) -> tuple[float, bool]: """ Compute a novelty score for an incoming perception. Args: embedding: Current embedding (any L2 norm; will be normalised). entities: Extracted entity strings for entity-novelty component. prediction_error: RMSE from PredictiveCodingModel (None = not available). Returns: (score: float in [0.0, 1.0], is_high_salience: bool) """ emb_score = self._embedding_novelty(embedding) entity_score = self._entity_novelty(entities or []) composite = min(1.0, 0.7 * emb_score + 0.3 * entity_score) if CONFIG['use_predictive_coding'] and prediction_error is not None: pe_weight = CONFIG['predictive_error_weight'] normalised_pe = min(1.0, prediction_error / self._PREDICTION_ERROR_SCALE) composite = min(1.0, (1.0 - pe_weight) * composite + pe_weight * normalised_pe ) high = composite >= self._threshold logger.debug( f"[SALIENCE:{self.subject_id}] " f"emb={emb_score:.3f} entity={entity_score:.3f} " f"composite={composite:.3f} high={high}" ) return composite, high def update(self, embedding: np.ndarray, entities: list[str] | None = None) -> None: """Add the current perception to the rolling window.""" norm = embedding / (np.linalg.norm(embedding) + 1e-8) self._embedding_window.append(norm) self._entity_window.append(set(entities or [])) # ── Internal helpers ───────────────────────────────────────────────────────── def _embedding_novelty(self, embedding: np.ndarray) -> float: if not self._embedding_window: return 1.0 norm = embedding / (np.linalg.norm(embedding) + 1e-8) recent = np.stack(list(self._embedding_window)) sims = recent @ norm mean_sim = float(np.mean(sims)) return max(0.0, min(1.0, 1.0 - mean_sim)) def _entity_novelty(self, entities: list[str]) -> float: if not entities: return 0.0 if not self._entity_window: return 1.0 seen: set[str] = set().union(*self._entity_window) novel = [e for e in entities if e not in seen] return len(novel) / len(entities)

# schema/pattern_miner.py """ Better pattern mining for schema induction (v1.3). Provides two complementary algorithms that replace the naive entity-label grouping used in v1.2: 1. Graph community detection (Louvain / greedy modularity communities). Identifies structurally cohesive clusters of episodic nodes in the co-occurrence / causal graph. """ nx_graph = graph.networkx_graph() episodic_ids = {n.memory_id for n in episodic_nodes} # Build an undirected co-occurrence subgraph subgraph = nx.Graph() for u, v, data in nx_graph.edges(data=True): if (u in episodic_ids and v in episodic_ids and data.get('edge_type') in ('cooccurrence', 'causal')): subgraph.add_edge(u, v, weight=data.get('weight', 0.1)) if subgraph.number_of_nodes() < min_count: return [] try: communities = list(nx.algorithms.community.greedy_modularity_communities( subgraph, weight='weight' )) except Exception as exc: logger.warning(f"[PATTERN_MINER] Community detection failed: {exc}") return [] id_to_node = {n.memory_id: n for n in episodic_nodes} patterns: list[tuple[str, list[EpisodicNode]]] = [] for community in communities: members = [id_to_node[mid] for mid in community if mid in id_to_node] if len(members) < min_count: continue # Find the dominant outcome label in this community outcome_counts: dict[str, int] = defaultdict(int) for node in members: lbl = _outcome_label(node.text) if lbl: outcome_counts[lbl] += 1 if not outcome_counts: continue dominant_outcome = max(outcome_counts, key=outcome_counts.__getitem__) # Find the dominant action entity action_counts: dict[str, int] = defaultdict(int) for node in members: for ae in _action_entities(node): action_counts[ae] += 1 if not action_counts: continue dominant_action = max(action_counts, key=action_counts.__getitem__) key = f"{dominant_action}:{dominant_outcome}" filtered = [ n for n in members if _outcome_label(n.text) == dominant_outcome and dominant_action in _action_entities(n) ] if len(filtered) >= min_count: patterns.append((key, filtered)) return patterns # ── Sequential pattern mining ───────────────────────────────────────────────────── def _sequential_patterns( episodic_nodes: list[EpisodicNode], min_count: int, ) -> list[tuple[str, list[EpisodicNode]]]: """ Mine frequent action subsequences from temporally ordered episodic nodes. """ max_len = CONFIG['pattern_max_length'] # Sort by creation time sorted_nodes = sorted(episodic_nodes, key=lambda n: n.created_at) # Build sequences of action-entity sets per node sequences: list[tuple[EpisodicNode, frozenset[str]]] = [] for node in sorted_nodes: actions = frozenset(_action_entities(node)) outcome = _outcome_label(node.text) if actions and outcome: sequences.append((node, actions, outcome)) if not sequences: return [] # Mine frequent singletons first singleton_counts: dict[tuple, list[EpisodicNode]] = defaultdict(list) for node, actions, outcome in sequences: for action in actions: key = (action, outcome) singleton_counts[key].append(node) patterns: list[tuple[str, list[EpisodicNode]]] = [] for (action, outcome), nodes in singleton_counts.items(): if len(nodes) >= min_count: patterns.append((f"{action}:{outcome}", nodes)) # Mine bigrams (consecutive pairs) if max_len >= 2: bigram_counts: dict[tuple, list[EpisodicNode]] = defaultdict(list) for i in range(len(sequences) - 1): node_a, actions_a, outcome_a = sequences[i] node_b, actions_b, outcome_b = sequences[i + 1] for aa in actions_a: for ab in actions_b: if aa != ab: key = (aa, ab, outcome_b) bigram_counts[key].append(node_b) for (aa, ab, outcome), nodes in bigram_counts.items(): if len(nodes) >= min_count: patterns.append((f"{aa}->{ab}:{outcome}", nodes)) return patterns # ── Public interface ────────────────────────────────────────────────────────────── class PatternMiner: """ Combines graph community detection and sequential pattern mining to produce a unified list of (pattern_key, node_list) pairs for schema induction. Returns: Sorted list of (pattern_key, nodes) with duplicates merged. """ min_count = CONFIG['schema_min_pattern_count'] episodic_nodes = self._graph.get_by_memory_type('episodic') community_results = _community_patterns(self._graph, episodic_nodes, min_count) sequential_results = _sequential_patterns(episodic_nodes, min_count) # Merge by key (union node lists, deduplicate by memory_id) merged: dict[str, dict[str, EpisodicNode]] = defaultdict(dict) for key, nodes in community_results + sequential_results: for node in nodes: merged[key][node.memory_id] = node result = [ (key, list(node_map.values())) for key, node_map in merged.items() if len(node_map) >= min_count ] result.sort(key=lambda x: len(x[1]), reverse=True) logger.info( f"[PATTERN_MINER:{self.subject_id}] " f"community={len(community_results)} sequential={len(sequential_results)} " f"merged={len(result)} patterns" ) return result

Design Notes — What Changed in v1.3

1. Hierarchical Schemas

SchemaInducer.detect() now runs in two phases. The flat induction phase (identical to v1.2, but powered by PatternMiner) produces level-1 procedural schema nodes. The hierarchical composition phase (_hierarchical_composition()) then groups these by shared action prefix and promotes groups of ≥ schema_compose_min_count (default: 2) into level-2 meta_schema nodes connected via composed_of edges. Additional hierarchy levels are supported up to schema_hierarchy_levels (default: 2). Meta-schema nodes are also of type procedural and are immune to pruning.

2. Precondition Learning

schema/precondition.py introduces PreconditionLearner, which runs as a post-induction pass in stack.consolidate(). For each procedural schema node, it walks the schema_of predecessor nodes and traverses their causal predecessor chains up to precondition_hop_depth (default: 2) hops. Entity strings appearing in ≥ precondition_min_freq (default: 0.6) of chains are stored in EpisodicNode.preconditions as a frequency dict. Preconditions are surfaced in recall results and Prometheus metrics.

3. Predictive Coding Integration

models/predictive.py adds PredictiveCodingModel, a two-layer online MLP that learns to predict the next embedding from the current one (single Adam gradient step per call). Prediction error (RMSE) is emitted as a Prometheus histogram and passed to the salience detector as a third signal weighted by predictive_error_weight (default: 0.3). The model is per-subject and lazy-loaded. Disable with USE_PREDICTIVE_CODING=false.

4. Multi-Agent / Social Memory

EpisodicNode gains source_agent (string ID, default 'self') and observation_type ('self' | 'observed' | 'communicated'). Both are propagated from metadata through the pipeline and exposed on all REST endpoints. MemoryGraph gains get_by_agent() and get_by_observation_type() query methods. RecallEngine.recall() gains a source_agent filter parameter. FeedbackIngester.record() forwards both fields. Social memories from non-self agents increment the SOCIAL_MEMORY_INGESTED Prometheus counter.

5. Active Recall / Dream Phase

When dream_active_recall=True (default), stack._dream_phase() replaces the passive edge-strengthening loop with an active recall pass. It selects causal nodes in the bottom strength quartile that have ≥ dream_min_causal_successors (default: 2) causal successors — these are structurally important (connected) but memory-weak (decaying). For each selected node, RecallEngine.dream_recall() runs a recall query using the node's text, and if the node appears in the top-k results, its strength is increased by dream_hit_strength_boost (default: 0.05) and its outgoing causal edges are strengthened. This prevents structurally important procedural memories from silently fading below the pruning threshold.

6. Better Pattern Mining

schema/pattern_miner.py introduces PatternMiner, which combines two algorithms. Graph community detection uses networkx.algorithms.community.greedy_modularity_communities (an approximation of Louvain) on the undirected co-occurrence/causal subgraph of episodic nodes to identify structurally dense clusters that likely represent recurring procedures. Sequential pattern mining scans temporally ordered node action-entity sequences for frequent singletons and bigrams (PrefixSpan-style). Both feeds are merged by pattern key (deduplicating by memory_id within each key) before schema promotion. Disable with USE_PATTERN_MINER=false to fall back to the v1.2 naive entity-label grouping.
